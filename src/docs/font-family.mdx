## Personalizando tu tema

<CustomizingYourTheme
  utility="font"
  name="font family"
  customName="display"
  customValue='"Oswald", "sans-serif"'
>

También puedes proporcionar valores predeterminados de `font-feature-settings` y `font-variation-settings` para una familia de fuentes:

```css
@theme {
  --font-display: "Oswald", "sans-serif";
  --font-display--font-feature-settings: "cv02", "cv03", "cv04", "cv11"; /* [!code highlight] */
  --font-display--font-variation-settings: "opsz" 32; /* [!code highlight] */
}
```

Si es necesario, utiliza la regla-at [@font-face](https://developer.mozilla.org/en-US/docs/Web/CSS/@font-face) para cargar fuentes personalizadas:

```css
@font-face {
  font-family: Oswald;
  font-style: normal;
  font-weight: 200 700;
  font-display: swap;
  src: url("/fonts/Oswald.woff2") format("woff2");
}
```

Si estás cargando una fuente de un servicio como [Google Fonts](https://fonts.google.com/), asegúrate de poner el `@import` al principio de tu archivo CSS:

```css
@import url("https://fonts.googleapis.com/css2?family=Roboto&display=swap"); /* [!code highlight] */
@import "tailwindcss";

@theme {
  --font-roboto: "Roboto", sans-serif; /* [!code highlight] */
}
```

Los navegadores requieren que las declaraciones `@import` vengan antes de cualquier otra regla, por lo que las importaciones de URL deben estar por encima de importaciones como `@import "tailwindcss"` que se insertan en el CSS compilado.
