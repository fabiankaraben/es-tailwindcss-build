Cada clase de utilidad en Tailwind puede aplicarse _condicionalmente_ añadiendo una variante al principio del nombre de la clase que describe la condición a la que quieres apuntar.

Por ejemplo, para aplicar la clase `bg-sky-700` en hover, usa la clase `hover:bg-sky-700`:

<Figure hint="Pasa el cursor sobre este botón para ver el cambio de color de fondo">

<Example>
  {
    <div className="grid place-items-center">
      <button className="rounded-full bg-sky-500 px-5 py-2 text-sm leading-5 font-semibold text-white hover:bg-sky-700">
        Guardar cambios
      </button>
    </div>
  }
</Example>

```html
<!-- [!code classes:hover:bg-sky-700] -->
<button class="bg-sky-500 hover:bg-sky-700 ...">Guardar cambios</button>
```

</Figure>

<details className="mb-10 rounded-xl border border-gray-200 px-6 py-3 prose open:pb-5 dark:prose-dark dark:border-gray-800">
<summary className="font-medium cursor-default select-none text-gray-900 dark:text-gray-200">¿Cómo se compara esto con el CSS tradicional?</summary>

Cuando se escribe CSS de la manera tradicional, un solo nombre de clase haría cosas diferentes según el estado actual:

<TipBad>{<>Tradicionalmente, el mismo nombre de clase aplica diferentes estilos en hover</>}</TipBad>

```css
.btn-primary {
  background-color: #0ea5e9;
}

.btn-primary:hover {
  background-color: #0369a1;
}
```

En Tailwind, en lugar de añadir los estilos para un estado hover a una clase existente, añades otra clase al elemento que _solo_ hace algo en hover:

<TipGood>{<>En Tailwind, se utilizan clases separadas para el estado predeterminado y el estado hover</>}</TipGood>

```css
.bg-sky-500 {
  background-color: #0ea5e9;
}

.hover\:bg-sky-700:hover {
  background-color: #0369a1;
}
```

¿Te das cuenta de cómo `hover:bg-sky-700` _solo_ define estilos para el estado `:hover`? No hace nada por defecto, pero tan pronto como pasas el cursor sobre un elemento con esa clase, el color de fondo cambiará a `sky-700`.

Esto es lo que queremos decir cuando decimos que una clase de utilidad puede aplicarse _condicionalmente_ — usando variantes puedes controlar exactamente cómo se comporta tu diseño en diferentes estados, sin salir nunca de tu HTML.

</details>

Tailwind incluye variantes para casi todo lo que necesitarás, incluyendo:

- [Pseudo-clases](#pseudo-classes), como `:hover`, `:focus`, `:first-child` y `:required`
- [Pseudo-elementos](#pseudo-elements), como `::before`, `::after`, `::placeholder` y `::selection`
- [Media queries y feature queries](#media-and-feature-queries), como breakpoints Responsive, modo oscuro y `prefers-reduced-motion`
- [Selectores de atributos](#attribute-selectors), como `[dir="rtl"]` y `[open]`
- [Selectores hijos](#child-selectors), como `& > *` y `& *`

Estas variantes incluso se pueden apilar para apuntar a situaciones más específicas, por ejemplo, cambiando el color de fondo en modo oscuro, en el breakpoint mediano, en hover:

```html
<!-- [!code classes:dark:md:hover:bg-fuchsia-600] -->
<button class="dark:md:hover:bg-fuchsia-600 ...">Guardar cambios</button>
```

En esta guía aprenderás sobre cada variante disponible en el framework, cómo usarlas con tus propias clases personalizadas e incluso cómo crear las tuyas propias.

## Pseudo-clases

### :hover, :focus, y :active

Da estilo a los elementos en hover, focus y active usando las variantes `hover`, `focus` y `active`:

<Figure hint="Intenta interactuar con este botón para ver los estados hover, focus y active">

<Example>

      >
        Guardar cambios
      </button>
    </div>
  }
</Example>

```html
<!-- [!code classes:hover:bg-violet-600,focus:outline-2,focus:outline-offset-2,focus:outline-violet-500,active:bg-violet-700] -->
<!-- prettier-ignore -->

  Guardar cambios
</button>
```

</Figure>

Tailwind también incluye variantes para otros estados interactivos como `:visited`, `:focus-within`, `:focus-visible` y más.

```html
<div class="nth-3:underline">
```

#### Grupos arbitrarios

Puedes crear variantes `group-*` únicas al vuelo proporcionando tu propio selector como un [valor arbitrario](/docs/adding-custom-styles#using-arbitrary-values) entre corchetes:

<CodeExampleGroup
  filenames={["HTML", "Generated CSS"]}
>
  <CodeBlock
    example={html`
      <!-- [!code classes:group-[.is-published]:block] -->
      <div class="group is-published">
        <div class="hidden group-[.is-published]:block">
          Publicado
        </div>
      </div>
    `}
  />
  <CodeBlock
    example={css`
      .group-\[\.is-published\]\:block {
        &:is(:where(.group):is(.is-published) *) {
          display: block;
        }
      }
    `}
  />
</CodeExampleGroup>

Para mayor control, puedes usar el carácter `&` para marcar dónde debe terminar `.group` en el selector final relativo al selector que estás pasando:

<CodeExampleGroup
  filenames={["HTML", "Generated CSS"]}
>
  <CodeBlock
    example={html`
      <!-- [!code classes:_&] -->
      <div class="group">
        <div class="group-[:nth-of-type(3)_&]:block">
          <!-- ... -->
        </div>
      </div>
    `}
  />
  <CodeBlock
    example={css`
      .group-\[\:nth-of-type\(3\)_\&\]\:block {
        &:is(:nth-of-type(3) :where(.group) *) {
          display: block;
        }
      }
    `}
  />
</CodeExampleGroup>

#### Grupos implícitos

La variante `in-*` funciona de manera similar a `group` excepto que no necesitas agregar `group` al elemento padre:

{/* prettier-ignore */}
```html
<!-- [!code classes:in-focus:opacity-100] -->
<!-- [!code --:3] -->
<div tabindex="0" class="group">
  <div class="opacity-50 group-focus:opacity-100">
<!-- [!code ++:3] -->
<div tabindex="0">
  <div class="opacity-50 in-focus:opacity-100">
    <!-- ... -->
  </div>
</div>
```

La variante `in-*` responde a cambios de estado en cualquier padre, así que si quieres un control más detallado, necesitarás usar `group` en su lugar.

## Consultas de medios y características

### Breakpoints Responsive

Para estilizar un elemento en un breakpoint específico, usa variantes responsive como `md` y `lg`.

Por ejemplo, esto renderizará una cuadrícula de 3 columnas en móvil, una cuadrícula de 4 columnas en pantallas de ancho medio, y una cuadrícula de 6 columnas en pantallas de ancho grande:

```html
<!-- [!code classes:md:grid-cols-4,lg:grid-cols-6] -->
<div class="grid grid-cols-3 md:grid-cols-4 lg:grid-cols-6">
  <!-- ... -->
</div>
```

Para estilizar un elemento basado en el ancho de un elemento padre en lugar del viewport, usa variantes como `@md` y `@lg`:

```html
<!-- [!code classes:@container,@md:flex-row] -->
<div class="@container">
  <div class="flex flex-col @md:flex-row">
    <!-- ... -->
  </div>
</div>
```

Consulta la documentación de [Responsive design](/docs/responsive-design) para una mirada en profundidad sobre cómo funcionan estas características.

### @supports

Usa la variante `supports-[...]` para aplicar estilos a las cosas basados en si una característica específica es compatible con el navegador del usuario:

```html
<!-- [!code classes:supports-[display:grid]:grid] -->
<div class="flex supports-[display:grid]:grid ...">
  <!-- ... -->
</div>
```

Bajo el capó, la variante `supports-[...]` genera [`@supports rules`](https://developer.mozilla.org/en-US/docs/Web/CSS/@supports) y toma cualquier cosa que usarías con `@supports (...)` entre los corchetes, como un par propiedad/valor, e incluso expresiones usando `and` y `or`.

Para brevedad, si solo necesitas verificar si una propiedad es compatible (y no un valor específico), puedes especificar solo el nombre de la propiedad:

```html
<!-- [!code classes:supports-backdrop-filter:bg-black/25,supports-backdrop-filter:backdrop-blur] -->
<div class="bg-black/75 supports-backdrop-filter:bg-black/25 supports-backdrop-filter:backdrop-blur ...">
  <!-- ... -->
</div>
```

Usa la variante `not-supports-[...]` para aplicar estilos a las cosas basados en si una característica específica no es compatible con el navegador del usuario:

```html
<!-- [!code classes:not-supports-[display:grid]:flex] -->
<div class="not-supports-[display:grid]:flex">
  <!-- ... -->
</div>
```

Puedes configurar atajos para reglas `@supports` comunes que estés usando en tu proyecto creando una nueva variante en el namespace `supports-*`:

```css
@custom-variant supports-grid {
  @supports (display: grid) {
    @slot;
  }
}
```

Luego puedes usar estas variantes personalizadas `supports-*` en tu proyecto:

```html
<!-- [!code classes:supports-grid:grid] -->
<div class="supports-grid:grid">
  <!-- ... -->
</div>
```

### Atributos de datos

Usa la variante `data-*` para aplicar estilos condicionalmente basados en [atributos de datos](https://developer.mozilla.org/en-US/docs/Learn/HTML/Howto/Use_data_attributes).

Para verificar si existe un atributo de datos (y no un valor específico), puedes especificar solo el nombre del atributo:

```html
<!-- Se aplicará -->
<!-- [!code classes:data-active:border-purple-500] -->
<div data-active class="border border-gray-300 data-active:border-purple-500">
  <!-- ... -->
</div>

<!-- No se aplicará -->
<div class="border border-gray-300 data-active:border-purple-500">
  <!-- ... -->
</div>
```

Si necesitas verificar un valor específico, puedes usar un valor arbitrario:

```html
<!-- Se aplicará -->
<!-- [!code classes:data-[size=large]:p-8] -->
<div data-size="large" class="data-[size=large]:p-8">
  <!-- ... -->
</div>

<!-- No se aplicará -->
<div data-size="medium" class="data-[size=large]:p-8">
  <!-- ... -->
</div>
```

Alternativamente, puedes configurar atajos para atributos de datos comunes que estés usando en tu proyecto creando una nueva variante en el namespace `data-*`:

```css
/* [!code filename:app.css] */
@import "tailwindcss";

@custom-variant data-checked (&[data-ui~="checked"]);
```

Luego puedes usar estas variantes personalizadas `data-*` en tu proyecto:

```html
<!-- [!code classes:data-checked:underline] -->
<div data-ui="checked active" class="data-checked:underline">
  <!-- ... -->
</div>
```

## Variantes personalizadas

### Usando variantes arbitrarias

Así como los [valores arbitrarios](/docs/adding-custom-styles#using-arbitrary-values) te permiten usar valores personalizados con tus clases de utilidad, las variantes arbitrarias te permiten escribir variantes de selector personalizadas directamente en tu HTML.

Las variantes arbitrarias son solo cadenas de formato que representan el selector, envueltas en corchetes. Por ejemplo, esta variante arbitraria cambia el cursor a `grabbing` cuando el elemento tiene la clase `is-dragging`:

<CodeExampleGroup
  filenames={["HTML", "CSS Generado"]}
>
  <CodeBlock
    example={svelte`
      <!-- [!code classes:[&.is-dragging]:cursor-grabbing] -->
      <ul role="list">
        {#each items as item}
          <li class="[&.is-dragging]:cursor-grabbing">{item}</li>
        {/each}
      </ul>
    `}

/>

  <CodeBlock
    example={css`
      .\[\&\.is-dragging\]\:cursor-grabbing {
        &.is-dragging {
          cursor: grabbing;
        }
      }
    `}
  />
</CodeExampleGroup>

Las variantes arbitrarias se pueden apilar con variantes incorporadas o entre sí, al igual que el resto de las variantes en Tailwind:

<CodeExampleGroup
  filenames={["HTML", "CSS Generado"]}
>
  <CodeBlock
    example={svelte`
      <!-- [!code classes:[&.is-dragging]:active:cursor-grabbing] -->
      <ul role="list">
        {#each items as item}
          <li class="[&.is-dragging]:active:cursor-grabbing">{item}</li>
        {/each}
      </ul>
    `}
  />
  <CodeBlock
    example={css`
      .\[\&\.is-dragging\]\:active\:cursor-grabbing {
        &.is-dragging {
          &:active {
            cursor: grabbing;
          }
        }
      }
    `}
  />
</CodeExampleGroup>

Si necesitás espacios en tu selector, podés usar un guion bajo. Por ejemplo, esta variante arbitraria selecciona todos los elementos `p` dentro del elemento donde agregaste la clase:

<CodeExampleGroup
  filenames={["HTML", "CSS Generado"]}
>
  <CodeBlock
    example={html`
      <!-- [!code classes:[&_p]:mt-4] -->
      <div class="[&_p]:mt-4">
        <p>Lorem ipsum...</p>
        <ul>
          <li>
            <p>Lorem ipsum...</p>
          </li>
          <!-- ... -->
        </ul>
      </div>
    `}
  />
  <CodeBlock
    example={css`
      .\[\&_p\]\:mt-4 {
        & p {
          margin-top: calc(var(--spacing) * 4);
        }
      }
    `}
  />
</CodeExampleGroup>

También podés usar at-rules como `@media` o `@supports` en variantes arbitrarias:

<CodeExampleGroup
  filenames={["HTML", "CSS Generado"]}
>
  <CodeBlock
    example={html`
      <!-- [!code classes:[@supports(display:grid)]:grid] -->
      <div class="flex [@supports(display:grid)]:grid">
        <!-- ... -->
      </div>
    `}
  />
  <CodeBlock
    example={css`
      .\[\@supports\(display\:grid\)\]\:grid {
        @supports (display:grid) {
          display: grid;
        }
      }
    `}
  />
</CodeExampleGroup>

Con las variantes personalizadas de at-rule, el marcador de posición `&` no es necesario, al igual que cuando se anida con un preprocesador.

### Registrando una variante personalizada

Si te encontrás usando la misma variante arbitraria varias veces en tu proyecto, podría valer la pena crear una variante personalizada usando la directiva `@custom-variant`:

```css
@custom-variant theme-midnight (&:where([data-theme="midnight"] *));
```

Ahora podés usar la variante `theme-midnight:<utility>` en tu HTML:

```html
<!-- [!code classes:theme-midnight:bg-black] -->
<html data-theme="midnight">
  <button class="theme-midnight:bg-black ..."></button>
</html>
```

Aprendé más sobre cómo agregar variantes personalizadas en la [documentación de agregar variantes personalizadas](/docs/adding-custom-styles#adding-custom-variants).

### Referencia de pseudo-clases

Esta es una lista completa de ejemplos para todas las variantes de pseudo-clases incluidas en Tailwind para complementar la [documentación de pseudo-clases](/docs/hover-focus-and-other-states#pseudo-classes) al inicio de esta guía.

#### :hover

Estiliza un elemento cuando el usuario pasa el cursor sobre él con el mouse usando la variante `hover`:

```html
<!-- [!code classes:hover:bg-white] -->
<div class="bg-black hover:bg-white ...">
  <!-- ... -->
</div>
```

#### :focus

Estiliza un elemento cuando tiene foco usando la variante `focus`:

```html
<!-- [!code classes:focus:border-blue-400] -->
<input class="border-gray-300 focus:border-blue-400 ..." />
```

#### :focus-within

Estiliza un elemento cuando él o uno de sus descendientes tiene foco usando la variante `focus-within`:

```html
<!-- [!code classes:focus-within:shadow-lg] -->
<div class="focus-within:shadow-lg ...">
  <input type="text" />
</div>
```

#### :focus-visible

Estiliza un elemento cuando ha sido enfocado usando el teclado con la variante `focus-visible`:

```html
<!-- [!code classes:focus-visible:outline-2] -->
<button class="focus-visible:outline-2 ...">Enviar</button>
```

#### :active

Estiliza un elemento cuando está siendo presionado usando la variante `active`:

```html
<!-- [!code classes:active:bg-blue-600] -->
<button class="bg-blue-500 active:bg-blue-600 ...">Enviar</button>
```

#### :visited

Estiliza un enlace cuando ya ha sido visitado usando la variante `visited`:

```html
<!-- [!code classes:visited:text-purple-600] -->
<a href="https://seinfeldquotes.com" class="text-blue-600 visited:text-purple-600 ..."> Inspiración </a>
```

#### :target

Estiliza un elemento si su ID coincide con el fragmento de URL actual usando la variante `target`:

```html
<!-- [!code classes:target:shadow-lg] -->
<div id="about" class="target:shadow-lg ...">
  <!-- ... -->
</div>
```

#### :first-child

Estiliza un elemento si es el primer hijo usando la variante `first`:

```svelte
<!-- [!code classes:first:pt-0] -->
<ul>
  {#each people as person}
    <li class="py-4 first:pt-0 ...">
      <!-- ... -->
    </li>
  {/each}
</ul>
```

#### :last-child

Estiliza un elemento si es el último hijo usando la variante `last`:

```svelte
<!-- [!code classes:last:pb-0] -->
<ul>
  {#each people as person}
    <li class="py-4 last:pb-0 ...">
      <!-- ... -->
    </li>
  {/each}
</ul>
```

#### :only-child

Estiliza un elemento si es el único hijo usando la variante `only`:

```svelte
<!-- [!code classes:only:py-0] -->
<ul>
  {#each people as person}
    <li class="py-4 only:py-0 ...">
      <!-- ... -->
    </li>
  {/each}
</ul>
```

#### :nth-child(odd)

Estiliza un elemento si es un hijo con número impar usando la variante `odd`:

```svelte
<!-- [!code classes:odd:bg-gray-100] -->
<table>
  {#each people as person}
    <tr class="bg-white odd:bg-gray-100 ...">
      <!-- ... -->
    </tr>
  {/each}
</table>
```

#### :nth-child(even)

Estiliza un elemento si es un hijo con número par usando la variante `even`:

```svelte
<!-- [!code classes:even:bg-gray-100] -->
<table>
  {#each people as person}
    <tr class="bg-white even:bg-gray-100 ...">
      <!-- ... -->
    </tr>
  {/each}
</table>
```

#### :first-of-type

Estiliza un elemento si es el primer hijo de su tipo usando la variante `first-of-type`:

```svelte
<!-- [!code classes:first-of-type:ml-6] -->
<nav>
  <img src="/logo.svg" alt="Vandelay Industries" />
  {#each links as link}
    <a href="#" class="ml-2 first-of-type:ml-6 ...">
      <!-- ... -->
    </a>
  {/each}
</nav>
```

#### :last-of-type

Estiliza un elemento si es el último hijo de su tipo usando la variante `last-of-type`:

```svelte
<!-- [!code classes:last-of-type:mr-6] -->
```

#### :only-of-type

Estiliza un elemento si es el único hijo de su tipo usando la variante `only-of-type`:

#### :empty

Estiliza un elemento si no tiene contenido usando la variante `empty`:

```svelte
<!-- [!code classes:empty:hidden] -->
<ul>
  {#each people as person}
    <li class="empty:hidden ...">{person.hobby}</li>
  {/each}

```

#### :disabled

Estiliza un input cuando está deshabilitado usando la variante `disabled`:

```html
<!-- [!code classes:disabled:opacity-75] -->
<input class="disabled:opacity-75 ..." />
```

#### :enabled

Estiliza un input cuando está habilitado usando la variante `enabled`, muy útil cuando solo quieres aplicar otro estilo cuando un elemento no está deshabilitado:

```html
<!-- [!code classes:enabled:hover:border-gray-400] -->
<input class="enabled:hover:border-gray-400 disabled:opacity-75 ..." />
```

#### :checked

Estiliza un checkbox o botón de radio cuando está seleccionado usando la variante `checked`:

```html
<!-- [!code classes:checked:bg-blue-500] -->
<input type="checkbox" class="appearance-none checked:bg-blue-500 ..." />
```

#### :indeterminate

Estiliza un checkbox o botón de radio en un estado indeterminado usando la variante `indeterminate`:

```html
<!-- [!code classes:indeterminate:bg-gray-300] -->
<input type="checkbox" class="appearance-none indeterminate:bg-gray-300 ..." />
```

#### :default

Estiliza una opción, checkbox o botón de radio que era el valor por defecto cuando la página cargó inicialmente usando la variante `default`:

```html
<!-- [!code classes:default:outline-2] -->
<input type="checkbox" class="default:outline-2 ..." />
```

#### :optional

Estiliza un input cuando es opcional usando la variante `optional`:

```html
<!-- [!code classes:optional:border-red-500] -->
<input class="border optional:border-red-500 ..." />
```
