Tailwind CSS v3.0 tiene que ser nuestro lanzamiento más emocionante hasta la fecha, incluyendo mejoras como:

- [Just-in-Time, todo el tiempo](#just-in-time-all-the-time) — tiempos de compilación ultrarrápidos, variantes apilables,
  soporte para valores arbitrarios, mejor rendimiento del navegador y más.

  en fondos coloreados.
- [API de Scroll snap](#scroll-snap-api) — un conjunto completo y componible de utilidades para scroll snapping
  solo con CSS.
- [Layout Multi-columna](#multi-column-layout) — para que finalmente puedas construir ese periódico en línea con el que has
  estado soñando.
- [Estilizado de controles de formulario nativos](#native-form-control-styling) — haz que checkboxes, radio buttons y

  tu HTML.

  muy fáciles de añadir.

  oído hablar.

  navegador.

Además de un hermoso y flamante [sitio web de documentación](https://tailwindcss.com), cargado con contenido mejorado y ejemplos en cada página.

Para empezar a jugar con Tailwind CSS v3.0 hoy, obtén la última versión de npm:

```sh
npm install -D tailwindcss@latest postcss autoprefixer
```

...o dirígete a [Tailwind Play](https://play.tailwindcss.com) para probar las últimas características directamente en el navegador.

Tailwind CSS v3.0 es una nueva versión mayor del framework y hay algunos cambios menores que rompen la compatibilidad, pero hemos trabajado muy duro para hacer el proceso de actualización lo más fluido posible, y para la mayoría de los proyectos deberías poder instalar v3.0 sin hacer ningún cambio.

Por ejemplo, [Tailwind UI](https://tailwindui.com) es probablemente el proyecto Tailwind más grande del mundo y cada plantilla es totalmente compatible tanto con v2 como con v3 sin necesidad de cambios.

Para más detalles e instrucciones paso a paso sobre cómo migrar a v3.0, consulta la <Link href="https://v3.tailwindcss.com/docs/upgrade-guide">guía de actualización</Link>.

## Just-in-Time, todo el tiempo

En marzo introdujimos el nuevo <Link href="/blog/just-in-time-the-next-generation-of-tailwind-css">motor Just-in-Time</Link> que trajo enormes ganancias de rendimiento, desbloqueó nuevas y emocionantes características como <Link href="https://v3.tailwindcss.com/docs/adding-custom-styles#using-arbitrary-values">valores arbitrarios</Link>, e hizo que las configuraciones complejas de variantes fueran cosa del pasado.

<YouTubeVideo id="3O_3X7InOw8" />

En Tailwind CSS v3.0, el nuevo motor se ha vuelto estable y ha reemplazado al motor clásico, por lo que cada proyecto de Tailwind puede beneficiarse de estas mejoras de forma predeterminada.

---

## Todos los colores listos para usar

Antes del nuevo motor, siempre teníamos que tener cuidado con el tamaño del archivo CSS en desarrollo, y una de las mayores concesiones que tuvimos que hacer fue limitar cuidadosamente la paleta de colores.

En v3.0, cada color de la paleta de colores extendida está habilitado por defecto, incluyendo lime, cyan, sky, fuchsia, rose y cincuenta sombras de gray.

Consulta la <Link href="https://v3.tailwindcss.com/docs/customizing-colors">referencia de la paleta de colores</Link> para obtener más información.

## Modificador de impresión


## Modificadores Portrait y landscape

Usa los nuevos modificadores `portrait` y `landscape` para añadir estilos condicionalmente cuando el viewport está en una orientación específica:

```html
<div>
  <div class="portrait:hidden">
    <!-- ... -->
  </div>
  <div class="landscape:hidden">
    <p>Esta experiencia está diseñada para ser vista en landscape. Por favor, rota tu dispositivo para ver el sitio.</p>
  </div>
</div>
```

La <Link href="https://v3.tailwindcss.com/docs/hover-focus-and-other-states#viewport-orientation">documentación para esta característica</Link> tiene incluso menos contenido que esta sección de esta publicación.

## Propiedades arbitrarias

Esto podría ser ilegal, pero hemos hecho posible añadir CSS totalmente arbitrario que puedes combinar con modificadores como `hover`, `lg` y cualquier otro:

```html
<div class="[mask-type:luminance] hover:[mask-type:alpha]">
  <!-- ... -->
</div>
```

Esto es lo que los estilos inline quieren ser cuando crezcan. Obtén más información en la documentación de <Link href="https://v3.tailwindcss.com/docs/adding-custom-styles#arbitrary-properties">propiedades arbitrarias</Link>.

## Play CDN

No hay forma de hacer una compilación CDN basada en CSS sensata para Tailwind CSS v3.0, así que tuvimos que hacer algo diferente — construimos una biblioteca JavaScript.

{/* prettier-ignore */}
```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Ejemplo</title>
    <script src="https://cdn.tailwindcss.com/"></script> <!-- [!code ++] -->
  </head>
  <body>
    <!-- -->
  </body>
</html>
```

Añade esa etiqueta de script a cualquier documento HTML y podrás usar todas las características de Tailwind, directamente en el navegador. Está destinado solo para fines de desarrollo, pero seguro que es una forma divertida de construir pequeñas demos o experimentar con una nueva idea.

Obtén más información en la documentación de <Link href="https://v3.tailwindcss.com/docs/installation/play-cdn">Play CDN</Link>.
